### Discussions

- Designed for conversations that aren't necessarily tied to a code - such as QnA, ideas, or general feedback
- Support open, ongoing communication within a shared forum and can be public or private, depending on the repository's visibility


#### Enabling a discussion in a repo in GitHub

- Repo owners and those with write access can enable GitHub Discussions for a community on their public and private repos
- The visibility of a discussion inherits visibility from the repo they're created in
- When you first enable GitHub Discussions, you're prompted to configure a welcome post

1. Navigate to the main page of the repo
1. Select _Settings_
1. Scroll down to _Features_ sections and under _Discussions_, select _Setup discussions_
1. Under _Start a new discussion_, edit the template to match your community's tone and resources
1. Select _Start discussion_

#### Creating a new discussion in GitHub

- Any authenticated user who can view the repo can create a discussion in that repo
- For organization-level discussions, any user who can view the source repo can also create a discussion

1. Navigate to the main page of the repo
1. Select _Discussions_
1. On the right side of the page, select _New Discussions_
1. Select a discussion category by choosing _Get Started_
    All discussions must be placed in a category. Repo maintainers define these categories
1. Under _Discussions_ title enter a title for your discussion and under _Write_ enter the body of your discussion
1. Select _Start discussion_

#### Category of Discussions

- Each discussion category includes a unique name, emoji, and description to clarify its purpose
- Categories help maintainers organize how conversations are filtered
- They're customizable to help distinguish categories that are QnA or more open-ended

| Category | Purpose | Format |
| -------- | ------- | ------ |
| Announcements | Updates and news from project maintainers | Announcements |
| General | Anything and everything relevant to the project | Open-ended discussion |
| Ideas | Ideas to change or improve the project | Open-ended discussion |
| Polls | Polls with multiple options for the community to vote for and discuss | Polls |
| QnA | Questions for the community to answer, with a question/answer format | Question and Answer |
| Show and tell | Creations, experiments, or tests relevant to the project | Open-ended discussion |

Repo maintainers can pin important discussions to the top of the Discussions tab for better visibility

#### Making a comment as an anwer in GitHub

In a QnA style dicussion, you can mark a comment as the accepted answer to the original question

1. Navigate to the discussion
1. Locate the comment that best answers the original question
1. Select _Mark as answer_ below the comment

The comment will be highlighted, making it easy for others to find the solution quickly. You can also unmark an answer if needed

#### Converting a discussion to an issue in GitHub

If a discussion leads to a work that needs to be tracked, the discussion can be converted into an issue

1. Navigate to the discussion that is to be converted
1. Select the three-dots menu at the top-right of the discussion
1. Select convert to an issue
1. Confirm the conversion and select a repo to create the new issue in

This keep track of actionable work that originates from community conversations

#### Pinning a discussion in GitHub

You can pin important discussions to the top of the Discussions page for better visibility

1. Navigate to the discussion that needs to be pin
1. Select the three-dot menu at the top-right of the discussion
1. Select Pin discussion

Helpful for highlighting announcements, important questions, or ongoing topics the community should notice
