### Issues

- Used to track ideas, feedback, tasks, or bugs
- Issues can be created from:
    1. An item in a task list
    1. A note in a project
    1. A comment in an issue or pull requests
    1. A specific line of code
    1. An URL query

#### Creating an issue from a repo in GitHub

1. Navigate to the main page of the repo
1. Under the repo name, select _Issues_
1. Select _New Issue_
1. If the repo uses issue templates, next to the type of the issue you'd like to open select _Get started_ <br />
    If the type of the issue you'd like to open isn't included in the available options, select _Open a blank issue_ <br />
    If not using template, skip to the next step <br />
    If the repo uses issue forms, you'll see a structured form to fill out instead of a blank issue template. Issue forms allow maintainers to collect specific information in a standardized format
1. In the _Add a title_ field, enter a title for your issue
1. In the _Add a description_ field, type a description of your issue
1. If you're a project maintainer, you can assign the issue to someone, add it to a project board, associate it with a milestone, or apply a label
1. Select _Submit a new issue_

> _Note:_ Use labels, mentions, and reactions to manage collaboration effecively and increase issue visibility
